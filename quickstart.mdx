## Prerequisites

Before you begin integrating with the ClaimMind API, ensure you have the following:

*   **API Base URL**: The base URL for the ClaimMind API (e.g., `https://api.vm01-ext.claimmind.dev.keycenter.ai/cl-service`).
*   **Authentication Credentials**: Obtain your `username` and `password` for API authentication.

## Making Your First API Call

This section guides you through authenticating and making your first request to the ClaimMind API.

### Step 1: Obtain an Access Token

To interact with the ClaimMind API, you first need to authenticate and obtain an access token.

<Steps>
<Step title="Send a login request">
Make a `POST` request to the `/v1/auth/login` endpoint with your `username` and `password` to obtain an access token.

<CodeGroup>
```bash
API_BASE_URL="https://api.vm01-ext.claimmind.dev.keycenter.ai/cl-service"
USERNAME="your_username@example.com"
PASSWORD="your_password"

curl -X POST "${API_BASE_URL}/v1/auth/login" \
--header 'Content-Type: application/x-www-form-urlencoded' \
--data-urlencode "username=${USERNAME}" \
--data-urlencode "password=${PASSWORD}"
```

```python
import requests

API_BASE_URL = "https://api.vm01-ext.claimmind.dev.keycenter.ai/cl-service" # Replace with your actual API base URL
USERNAME = "your_username@example.com"
PASSWORD = "your_password"

response = requests.post(
    f"{API_BASE_URL}/v1/auth/login",
    headers={'Content-Type': 'application/x-www-form-urlencoded'},
    data={
        'username': USERNAME,
        'password': PASSWORD
    }
)
print(response.json())
```
</CodeGroup>

<Check>
A successful response will return an `access_token` and a `refresh_token`. Store the `access_token` securely, as it will be used for subsequent authenticated requests to the ClaimMind API.
</Check>
</Step>
</Steps>

### Step 2: List Users

Once you have an access token, you can start making authenticated requests. Let's try listing the available users.

<Steps>
<Step title="Send a request to list users">
Make a `GET` request to the `/v1/users` endpoint, including your `access_token` in the `Authorization` header.

<CodeGroup>
```bash
curl -X GET "{api_base_url}/v1/users" \
-H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

```python
import requests

api_base_url = "https://api.vm01-ext.claimmind.dev.keycenter.ai/cl-service" # Replace with your actual API base URL
access_token = "YOUR_ACCESS_TOKEN" # Replace with your obtained access token

response = requests.get(
    f"{api_base_url}/v1/users",
    headers={
        "Authorization": f"Bearer {access_token}"
    }
)
print(response.json())
```
</CodeGroup>

<Check>
A successful response will return a list of users in your tenant.
</Check>
</Step>
</Steps>

### Step 3: Create a Claim

Now, let's create a new medical claim. This requires your `X-Tenant-Id` header.

<Steps>
<Step title="Send a request to create a claim">
Make a `POST` request to the `/v1/claims/create` endpoint, including your `access_token` in the `Authorization` header, along with the claim details in the request body.

<CodeGroup>
```bash
curl -X POST "{api_base_url}/v1/claims/create" \
-H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
-H "Content-Type: application/json" \
-d '{
  "claim_id": "YOUR_CLAIM_ID",
  "claim_code": "CLM-20250824-001"
}'
```

```python
import requests
import uuid

api_base_url = "https://api.vm01-ext.claimmind.dev.keycenter.ai/cl-service" # Replace with your actual API base URL
access_token = "YOUR_ACCESS_TOKEN" # Replace with your obtained access token

claim_data = {
    "claim_id": str(uuid.uuid4()), # Replace with a unique claim ID
    "claim_code": "CLM-20250824-001"
}

response = requests.post(
    f"{api_base_url}/v1/claims/create",
    json=claim_data,
    headers={
        "Authorization": f"Bearer {access_token}",
        "Content-Type": "application/json"
    }
)
print(response.json())
```
</CodeGroup>

<Check>
A successful response will return the details of the newly created claim, including its unique `id`.
</Check>
</Step>
</Steps>
