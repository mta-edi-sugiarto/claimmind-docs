## Overview

This section provides a human-readable overview of the most critical data models used throughout the ClaimMind API. While the OpenAPI specification offers detailed schemas, this guide focuses on explaining the purpose of each model and its key fields, helping you understand the data you will be sending and receiving.

Understanding these data models is crucial for designing your applications to interact seamlessly with the ClaimMind API.

## Core Data Models

### AdminRegistrationCreate
Use this model to create a new administrator registration.

*   **`first_name` (String)**: The first name of the administrator.
*   **`last_name` (String)**: The last name of the administrator.
*   **`email` (String, Email)**: The email address of the administrator.
*   **`password` (String)**: The password for the administrator's account.
*   **`tenant_id` (UUID)**: The unique identifier of the tenant (organization) the administrator belongs to.
*   **`phone_number` (String)**: The phone number of the administrator.

### AdminRegistrationResponse
This model represents the response after a successful administrator registration.

*   **`id` (UUID)**: The unique identifier for the administrator.
*   **`name` (String)**: The full name of the administrator.
*   **`username` (String)**: The username for the administrator's account.
*   **`email` (String, Email)**: The email address of the administrator.
*   **`tenant_id` (UUID)**: The unique identifier of the tenant (organization) the administrator belongs to.

### CBGCbgPayload
Use this model to create a CBG workflow.

*   **`claim_id` (UUID)**: The unique identifier for the claim.
*   **`icd_code` (Object)**: The ICD code object.
*   **`ocr_result` (String)**: The OCR result in string format.
*   **`regional` (Integer, Optional)**: The regional identifier.
*   **`jenis_rumah_sakit` (String, Optional)**: The type of hospital.
*   **`kelas_rumah_sakit` (String, Optional)**: The class of hospital.
*   **`jenis_perawatan` (String, Optional)**: The type of care.
*   **`kelas_rawat` (Integer, Optional)**: The class of care.
*   **`nama_rumah_sakit_khusus` (String, Optional)**: The name of the special hospital.

### CBGResultPushRequest
Use this model to push CBG results to a claim.

*   **`claim_id` (UUID)**: The unique identifier for the claim.
*   **`workflow_id` (String, Optional)**: The ID of the workflow run.
*   **`cbg_code` (Object, Optional)**: The CBG code object.
*   **`full_json` (Object, Optional)**: The complete JSON object containing the CBG result data.

### CBGupdatePayload
Use this model to update a CBG workflow.

*   **`claim_id` (UUID)**: The unique identifier for the claim.
*   **`cbg_code` (Object, Optional)**: The CBG code object.

### ChecklistItemFull
This model represents a full checklist item.

*   **`id` (String, Optional)**: The unique identifier for the checklist item.
*   **`requirement` (String, Optional)**: The requirement description.

### ClaimValidationRow
This model represents a row in the claim validation table.

*   **`id` (Integer)**: The unique identifier for the validation row.
*   **`claim_id` (UUID, Optional)**: The unique identifier for the claim.
*   **`tenant_id` (UUID, Optional)**: The unique identifier for the tenant.
*   **`workflow_id` (String)**: The ID of the workflow run.
*   **`status` (String, Optional)**: The status of the validation.
*   **`validation_result` (Object)**: The detailed result of the validation.
*   **`version` (Integer, Optional)**: The version of the validation result.
*   **`created_at` (String, Date-Time, Optional)**: The timestamp when the record was created.
*   **`updated_at` (String, Date-Time, Optional)**: The timestamp when the record was last updated.

### ClaimValidationUpdateRequest
Use this model to update a claim validation result.

*   **`workflow_id` (String)**: The ID of the workflow run.
*   **`status` (String)**: The status of the validation (default: "completed").
*   **`validation_result` (Object)**: The detailed result of the validation.

### ClaimValidationUpdateResponse
This model represents the response after updating a claim validation.

*   **`updated` (Boolean)**: Indicates if the update was successful.
*   **`data` (ClaimValidationRow, Optional)**: The updated claim validation row.

### ClaimValidatorInput
This model defines the input data for triggering a claim validation workflow.

*   **`claim_id` (String)**: The identifier of the claim to validate.
*   **`tenant_id` (String)**: The identifier of the tenant.
*   **`claim_markdown` (String)**: The claim content in markdown format.
*   **`patient_profile` (PatientProfile)**: The patient's profile information.
*   **`validation_output` (Array of Objects, Optional)**: The output of the validation.
*   **`custom_rules` (Array of Objects, Optional)**: Custom validation rules.

### ClaimValidatorRequest
Use this model to trigger a claim validation workflow.

*   **`input_data` (ClaimValidatorInput)**: An object containing the data needed for validation.

### ClaimValidatorResponse
This model represents the response after triggering a claim validation.

*   **`claim_id` (String, Optional)**: The claim ID that was validated.
*   **`workflow_id` (String, Optional)**: The ID of the workflow run for this validation.
*   **`status` (String, Optional)**: The status of the validation process.
*   **`result` (Object, Optional)**: The detailed result of the claim validation.

### CodeItem
This model represents a code item, such as an ICD-10 or ICD-9 code.

*   **`code` (String)**: The code value.
*   **`description` (String, Optional)**: A description of the code.

### DatabaseType
This enum defines the supported database types.

*   **`turso` (String)**: Turso database type.
*   **`postgres` (String)**: PostgreSQL database type.

### HTTPValidationError
This model represents a standard HTTP validation error.

*   **`detail` (Array of ValidationError)**: A list of validation error details.

### ICDCodePayload
Use this model to submit ICD codes.

*   **`claim_id` (String)**: The unique identifier for the claim.
*   **`workflow_id` (String, Optional)**: The ID of the workflow run.
*   **`full_json` (Object)**: The complete JSON object containing the ICD code data.

### ICDResultPushRequest
Use this model to push ICD results to a claim.

*   **`claim_id` (UUID)**: The claim ID to which the ICD result belongs.
*   **`patient_profile` (PatientProfile)**: The patient's profile information.

### MemoryBlockRequest
Use this model to create a memory block.

*   **`claim_id` (UUID)**: The unique identifier for the claim.
*   **`label` (String)**: The label for the memory block.
*   **`markdown_text` (String)**: The content of the memory block in markdown format.

### MemoryBlockResponse
This model represents the response after creating a memory block.

*   **`workflow_id` (String)**: The ID of the workflow run.
*   **`claim_id` (UUID)**: The unique identifier for the claim.
*   **`status` (String)**: The status of the memory block creation.
*   **`message` (String)**: A descriptive message about the status.

### OCRClaimsRequest
Use this model to create a new claim record in the system.

*   **`claim_id` (String)**: A unique identifier for the claim.
*   **`claim_code` (String, Optional)**: A human-readable code for the claim.

### OCRFullRequest
Use this model to trigger a full OCR workflow.

*   **`claim_id` (UUID, Optional)**: The unique identifier for the claim.
*   **`base64_list` (Array of Strings, Optional)**: A list of base64 encoded strings of the documents to be processed.
*   **`filenames` (Array of Strings, Optional)**: A list of filenames associated with the documents.
*   **`raw_input` (String, Optional)**: Raw text input to be processed.

### OCRFullResponse
This model represents the response after triggering a full OCR workflow.

*   **`claim_id` (UUID)**: The unique identifier for the claim.
*   **`workflow_id` (String)**: The ID of the workflow run.
*   **`result` (String)**: The detailed result of the OCR processing.
*   **`version` (Integer)**: The version of the OCR result.

### OCRFullupdateresultRequest
Use this model to update the result of a full OCR workflow.

*   **`workflow_id` (String)**: The ID of the workflow run.
*   **`result` (String)**: The updated result of the OCR processing.
*   **`ocr_validation` (Object)**: The OCR validation object.

### OCRPagePayload
This model represents the payload for an OCR page.

*   **`document_id` (UUID, Optional)**: The unique identifier for the document.
*   **`page_number` (Integer)**: The page number.
*   **`raw_text` (String)**: The raw text extracted from the page.
*   **`category` (String)**: The category of the page.
*   **`ocr_validation` (Object, Optional)**: The OCR validation object for the page.

### OCRPageUpdateRequest
Use this model to update OCR pages.

*   **`workflow_id` (String)**: The ID of the workflow run.
*   **`pages` (Array of OCRPagePayload)**: A list of OCR page payloads.

### OCRRevisionItem
This model represents an item in an OCR revision.

*   **`page_number` (Integer)**: The page number.
*   **`raw_text` (String, Optional)**: The raw text of the page.
*   **`blocks` (Object, Optional)**: The blocks of text on the page.
*   **`category` (String, Optional)**: The category of the page.

### OCRRevisionRequest
Use this model to create an OCR revision.

*   **`claim_id` (UUID)**: The unique identifier for the claim.
*   **`pages` (Array of OCRRevisionItem)**: A list of OCR revision items.
*   **`full_text_markdown` (String, Optional)**: The full text in markdown format.
*   **`category` (String, Optional)**: The category of the revision (default: "full_markdown").

### OCRRevisionResponse
This model represents the response after creating an OCR revision.

*   **`claim_id` (UUID)**: The unique identifier for the claim.
*   **`version` (Integer)**: The version of the OCR revision.
*   **`inserted` (Integer)**: The number of inserted items.
*   **`status` (String)**: The status of the revision (e.g., "ok", "not_found").

### OCRTriggerRequest
Use this model to trigger an OCR workflow for a claim, typically for PII extraction.

*   **`claim_id` (String)**: The identifier of the claim to process.
*   **`claim_code` (String, Optional)**: A human-readable code for the claim.
*   **`base64` (String, Optional)**: A base64 encoded string of the document to be processed.
*   **`raw_input` (String, Optional)**: Raw text input to be processed.
*   **`workflow_id` (String, Optional)**: An existing workflow ID to use.

### OCRTriggerResponse
This model represents the generic response structure when a workflow like claim creation or OCR is triggered.

*   **`claim_id` (UUID, Optional)**: The claim ID that was processed.
*   **`tenant_id` (UUID, Optional)**: The organization (tenant) to which this claim belongs.
*   **`status` (String, Optional)**: The status of the workflow.
*   **`message` (String, Optional)**: A descriptive message about the workflow status.
*   **`workflow_id` (String, Optional)**: The ID of the workflow run that was initiated.
*   **`result` (Object, Optional)**: The detailed result of the processing.

### OCRUpdateResponse
This model represents the response after updating an OCR.

*   **`updated` (Boolean)**: Indicates if the update was successful.
*   **`data` (OCRUpdateRow, Optional)**: The updated OCR row.

### OCRUpdateRow
This model represents a row in the OCR update table.

*   **`id` (UUID)**: The unique identifier for the OCR row.
*   **`workflow_id` (String, Optional)**: The ID of the workflow run.
*   **`claim_id` (UUID, Optional)**: The unique identifier for the claim.
*   **`raw_text` (String, Optional)**: The raw text extracted.
*   **`updated_at` (String, Date-Time, Optional)**: The timestamp when the record was last updated.
*   **`version` (Integer, Optional)**: The version of the OCR result.

## Relationships

*   A **Claim** is identified by a `claim_id` and is the central entity for workflows.
*   Workflows like **OCR**, **Validation**, **ICD Coding**, and **CBG** are triggered for a specific **Claim**.
*   **Users** belong to a **Tenant** (Organization).
*   Responses like **OCRTriggerResponse**, **ClaimValidatorResponse**, **OCRFullResponse**, and **MemoryBlockResponse** provide the status and `workflow_id` for the initiated processes.
*   **PatientProfile** is a crucial component for **ICDResultPushRequest** and **ValidationRulesRequest**, containing patient-specific medical codes.

This overview should provide a solid foundation for understanding the data structures within the ClaimMind API. For precise field definitions and constraints, always refer to the OpenAPI specification.
